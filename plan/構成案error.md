### 📑 構成案：Windows環境構築トラブルシューティング編

| 章 | 内容・ポイント |
| :--- | :--- |
| **タイトル** | 検索でヒットしやすいよう、具体的なエラー名やツール名を入れます。 |
| **はじめに** | 「Windowsでの開発はMac/Linuxと違って辛いことが多い」という共感から入り、「でも解決策はある」と提示します。 |
| **Case 1** | **PowerShellで `venv` が有効化できない**<br>あの赤いエラー画面（実行ポリシー）の解決策。あなたのメモをそのまま活かします。 |
| **Case 2** | **Dockerでホットリロードが効かない**<br>「ファイルを変えたのに反映されない」問題。`nodemon` の設定やポーリング（legacy-watch）について解説。 |
| **Case 3** | **Globalインストール vs Localインストール**<br>「どっちに入れればいいの？」という混乱を整理。今回は「Docker内」が正解である理由も補足。 |
| **Case 4** | **パスの区切り文字問題（`\` vs `/`）**<br>Windows特有のバックスラッシュ問題と、Git BashやWSL2を使うメリットに軽く触れる。 |
| **まとめ** | トラブルを解決したら、メイン記事で開発を進めよう！という誘導。 |
